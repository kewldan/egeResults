# Full code review — `ege_notifier`

_Дата: 2026-06-17. Полное ревью кодовой базы по пяти подсистемам (models/crypto,
services, bot, providers, config/lifecycle). Несущие выводы проверены по реальному
коду. Критических багов с потерей данных нет — недеструктивный `merge_results`
ограничивает радиус любого сбоя парсинга/фетча. Реальные проблемы —
в деплое и операционной устойчивости._

**Проверено вручную (важное):**
- ❌ **Отброшено как ложное:** заявленные «битые HTML-теги» (`<\b>` вместо `</b>`)
  в `texts.py` — **их нет** (grep по `<\[a-z]>` пуст; экранирование чистое).
- ✅ **Подтверждено:** `compose.yaml` не содержит сервиса `mongo` и не задаёт
  `MONGO_URI`, хотя комментарий в файле утверждает обратное.
- ✅ **Подтверждено:** `Notifier.send` рекурсивно ретраит `TelegramRetryAfter`
  без ограничения, хотя комментарий говорит «один раз».

## 🔴 High

### 1. Docker-деплой не достучится до MongoDB — `compose.yaml`
Нет сервиса `mongo`, а `bot.environment` переопределяет только `REDIS_URL`, не
`MONGO_URI`. Комментарий в шапке утверждает «MONGO_URI … переопределяют», но это
не так. С дефолтным `.env` (`mongodb://localhost:27017`) `localhost` внутри
контейнера — это сам контейнер → все операции с БД падают в рантайме, тогда как
старт логирует «MongoDB подключена».
**Fix:** добавить сервис `mongo` + `MONGO_URI: mongodb://mongo:27017` +
`depends_on` с healthcheck; либо, если Mongo внешняя (Atlas) — исправить
комментарий и README.

### 2. Небезопасные дефолты для прода — `config.py:27,86,88`
- `admin_id = 787751346` зашит в код (это ID владельца — ок для своего бота, но
  футган, если репозиторий шарят/используют как шаблон: все уведомления «новый
  пользователь / новый результат» уйдут этому ID).
- `identity_secret = "change-me-please"` и `encryption_key = None`
  (→ паспорта в открытом виде) только **предупреждают**. При крошечном
  пространстве паспортов (~10¹⁰) утечка БД с дефолтным секретом позволяет офлайн
  перебором восстановить паспортные цифры из `identity_hash`.
- **Fix:** жёстко падать на старте, если `provider == "ege_spb"` и любой из них
  оставлен дефолтным, а не только логировать.

## 🟠 Medium

### 3. `Cipher.decrypt` скармливает шифротекст в запрос паспорта при смене ключа — `security.py:63-77`
На `InvalidToken` возвращает исходное значение как есть. Если `ENCRYPTION_KEY`
сменили, вернётся шифротекст `gAAAAA…`, который `to_query` затем POST'ит на
ege.spb.ru как номер паспорта → сайт отвечает «не найдено» → `not_found=True` у
каждого затронутого ученика, маскируясь под опечатку.
**Fix:** бросать отдельное исключение, чтобы вызывающий пропустил фетч, а не
ходил на сайт с мусором.

### 4. `Notifier.send` ретраит `TelegramRetryAfter` без ограничения — `notifier.py:41-45` (та же форма в `ui.py:20-26`)
Комментарий говорит «пробуем снова один раз», но код рекурсирует без счётчика
попыток. Поскольку `broadcast` — последовательный цикл, один залипший чат в день
публикации может застопорить всю рассылку.
**Fix:** добавить лимит попыток и ограничить сон по `retry_after`.

### 5. Счётчик монитора может откатиться → ложный delta — `monitor.py` `_save`
`_save` перезаписывает `results_count` текущим снимком без проверки
монотонности (пропускает только когда новое значение `None`). Кратковременно
меньший счётчик уводит его назад, и следующий `diff_page` вычислит завышенный
положительный `delta` (переоценка «сколько сдавали», что управляет публичным
анонсом).
**Fix:** двигать только вперёд: обновлять, когда
`snapshot.results_count > state.results_count`.

### 6. `confirm_add` может бросить `KeyError` на отсутствующих FSM-данных — `add_student.py:81-90`
Читает `data["last_name"]` и т.п. напрямую. С `RedisStorage` (состояние
переживает рестарт) или при быстром двойном тапе на «Сохранить» (первый клик
делает `state.clear()` до чтения ключей) `get_data()` может не содержать ключей →
необработанное исключение.
**Fix:** читать через `.get()` и выходить, если поля нет; чистить состояние
после извлечения.

### 7. Парсер обзорной страницы позиционно хрупок — `ege_spb_overview.py:67,20-23`
Зашитый `#w2` предполагает, что «Основной период» всегда 2-я панель — семантическая
метка лежит в соседнем `.wave-name`, поэтому перестановка/добавление волны молча
переключит мониторинг на не тот период. Плюс жадная регулярка счётчика в fallback
(когда `.info-board` отсутствует) склеивает соседние числа (напр. год) в счётчик.
**Fix:** выбирать панель по тексту заголовка; ужесточить захват в fallback.

## 🟡 Low (стоит batch-уборкой)

- **Нет ping БД на старте** (`db.py`) — добавить `await client.admin.command("ping")`,
  чтобы простой Mongo падал сразу, а не пооперационно.
- **`value.isdigit()` ≠ парсится в int** (`ege_spb.py:94`) — `int("²")` бросает,
  хотя `"²".isdigit()` == `True`; использовать `value.isascii() and value.isdigit()`.
  Теоретично для реальных баллов, ловится ниже по стеку.
- **`mask_passport` игнорирует аргумент `series`** и раскрывает весь номер при
  `len < 2` (`security.py:23-27`) — только defense-in-depth (валидаторы требуют 6 цифр).
- **`build_form_body` может бросить `UnicodeEncodeError`** на не-cp1251 фамилии из
  сохранённых/мигрированных данных (`ege_spb.py:25-35`) — передать `errors="replace"`
  или предвалидировать.
- **Root-логгер на DEBUG может вытащить PII** — `httpcore`/`pymongo` не заглушены
  (`logging_setup.py`); на DEBUG они логируют тело POST с паспортными цифрами.
  Плюс `log_level` не валидируется → голый traceback на опечатке.
- **Секреты через целиком `env_file: .env`** — `ENCRYPTION_KEY`/`IDENTITY_SECRET`
  читаются через `docker inspect`; рассмотреть Docker secrets. Нет `healthcheck`
  у сервиса `bot`.
- **Нет реального фикстура «не найдено»** — направление not-found у `looks_not_found`
  тестируется только синтетической строкой; снять реальную страницу.
- **Нет лимита на `share:`** — подписчик может наплодить сколько угодно `ShareToken`
  (истекают, влияние низкое).

## ✅ Notably well done

- **tz-awareness сквозная** (`utcnow()` aware UTC + `tz_aware=True`) — самый частый
  футган Beanie/Mongo, обработан верно для TTL, cooldown и `expires_at`.
- **Дизайн share-токена** — хранится только `sha256(token)`, высокоэнтропийный
  секрет только в deep-link, атомарное одноразовое гашение через
  `find_one_and_delete` с фильтром `expires_at`.
- **`identity_hash` не зависит от `last_name`**, поэтому миграции фамилий не ломают
  дедуп; `normalize_digits` схлопывает различия форматирования.
- **Per-student lock + перечитывание под локом + `replace()` (не upsert)** реально
  закрывает окна lost-update и «воскрешения удалённого ученика», применено
  единообразно на success/error/not-found.
- **Авторизация на каждом callback с PII** (`student:`/`check:`/`share:`/`del:`) —
  подделанный callback не вытащит чужие баллы. Экранирование (`_esc`)
  дисциплинированное и полное; баллы в `_spoiler(_esc(...))`.
- **`StudentNotFoundError` не стирает существующие результаты**; плановый цикл
  глотает её, оставаясь живым; `merge_results` сохраняет предметы, отсутствующие
  в ответе.
- **Аккуратный жизненный цикл** в `__main__` — стартовые задачи удерживаются от GC,
  исключения логируются, упорядоченный `_safe_close` на shutdown; джобы планировщика
  `max_instances=1, coalesce=True`; монитор фиксирует состояние только после
  успешной рассылки (анонс ровно раз, никогда на свежем деплое).
